# 国内商品期货时序 CTA（5分钟K线、分到期合约拼接、双均线 Demo）回测规范文档

> 本文档仿照《Binance 合约 Reverse CTA（动量衰竭反转做空）回测规范文档》的写法与结构，面向 **国内商品期货** 场景，定义一个 **基于 5 分钟 K 线、先下载分到期日合约、再做收益率化换月拼接、最后基于时序规则进行回测** 的标准流程。模板参考原文档：[Source](https://www.genspark.ai/api/files/s/lW6nyFZh)
>
> 本项目聚焦的是：
> **在免费数据约束下，能否构建一条口径清晰、可复现、可持续扩展的国内商品期货 5 分钟时序 CTA 研究流水线，并给出一个可直接运行的双均线回测 Demo。**
>
> 本文档不是高频盘口策略，不是跨品种配对套利，也不是期限结构套利；它首先解决的是 **数据源可得性、主力换月连续化、回测口径统一** 三个问题。

## 一、连续合约构建规范（分到期合约 → 收益率拼接）

### 1.1 为什么不用简单价格硬拼接

国内商品期货不同交割月之间常有明显价格层级差异。若直接把 `RB2510` 的最后收盘价和 `RB2601` 的第一根收盘价首尾硬接，会把 **期限结构差、交割月差、升贴水差** 误当作真实价格跳空，从而污染：

- 均线；
- 趋势信号；
- 波动率；
- 回撤与收益归因。

因此，本规范默认采用 **收益率拼接**，而不是裸价格拼接。

### 1.2 默认拼接方法：收益率链式连续化

对于每个交割月合约 `c`：

```text
ret_t = close_t / close_{t-1} - 1
```

然后定义一个规范化连续价格 `norm_close`：

```text
初始基准价 = 1000
第一个合约内部：norm_close_t = 1000 × 累乘(1 + ret_t)
下一合约开始时：以前一合约最后一个 norm_close 作为新段起点
```

即：

```text
carry_0 = 1000
for contract in contracts_in_roll_order:
    contract.ret = contract.close.pct_change()
    contract.norm_close = carry_{prev} * cumprod(1 + contract.ret)
    carry_{next} = contract.norm_close[-1]
```

该方法保留了：

1. 单合约内部真实的涨跌节奏；
2. 换月时净值曲线的连续性；
3. 对趋势指标、均线、收益统计更友好的结构。

### 1.3 默认换月顺序

在免费 Demo 中，默认按 **交割月自然顺序** 连接：

```text
RB2510 -> RB2601 -> RB2605 -> RB2610 -> ...
```

正式版可升级为：

1. **按持仓量/成交量主力切换**；
2. **到期前 N 个交易日强制换月**；
3. **主力切换 + 流动性过滤联合判定**。

### 1.4 换月窗口处理建议

| 方法 | 说明 | 推荐度 |
|---|---|---|
| 首尾无重叠直接连接 | 适合 Demo，最简单 | 高 |
| 有重叠时按首个可交易时点切换 | 较稳妥 | 高 |
| 固定提前 N 天换月 | 更接近实盘 | 中 |
| 持仓量超过阈值时换月 | 更市场化 | 高 |
| 双边加权平滑切换 | 更高级，但实现更复杂 | 中 |

> **⚠ 若使用均线、趋势、ATR 等价格路径敏感指标，必须先做收益率拼接，再算指标。**

## 二、策略定义（Time-Series CTA 双均线 Demo）

### 2.1 策略目标

本 Demo 不追求“最优参数”，而是验证：

1. 免费分钟数据是否足以走通完整流程；
2. 换月连续化后的序列能否直接用于时序规则；
3. 回测引擎能否输出标准化指标、交易表、净值曲线。

因此，这里选用最透明的 **双均线趋势策略** 作为演示规则。

### 2.2 默认策略规则

设：

- `fast_ma = MA(norm_close, 24)`，对应约 2 小时；
- `slow_ma = MA(norm_close, 96)`，对应约 8 小时。

信号定义：

```text
if fast_ma > slow_ma: signal = +1
if fast_ma < slow_ma: signal = -1
else: signal = 0
```

持仓定义：

```text
position_t = signal_t
实际收益使用 position_{t-1} × return_t
```

即默认采用 **下一根 bar 开始生效** 的近似回测口径，避免同 bar 未来函数。

### 2.3 策略参数（Demo 默认值）

| 参数 | 默认值 | 含义 |
|---|---:|---|
| `period` | `5m` | K 线周期 |
| `fast_ma` | `24` | 快均线窗口 |
| `slow_ma` | `96` | 慢均线窗口 |
| `fee_bps_per_side` | `2` | 单边手续费（示例） |
| `slippage_bps_per_side` | `1` | 单边滑点（示例） |
| `round_trip_cost` | `6 bps` | 双边总成本 |
| `base_price` | `1000` | 连续化起始基准价 |

### 2.4 收益计算

策略逐 bar 收益定义：

```text
asset_ret_t    = norm_close_t / norm_close_{t-1} - 1
strategy_ret_t = position_{t-1} × asset_ret_t - cost_t
```

其中交易成本：

```text
trade_t = abs(position_t - position_{t-1})
cost_t  = trade_t × round_trip_cost / 2
```

### 2.5 Demo 回测口径

| 项目 | 口径 |
|---|---|
| 成交价近似 | bar close 生成信号，下一 bar 生效 |
| 杠杆 | Demo 中未单独建模保证金放大 |
| 成本 | 用固定 bps 扣减 |
| 仓位 | `-1 / 0 / +1` |
| 多空方向 | 双向都做 |
| 强平 | 样本最后一根统一平仓 |

> **⚠ 本 Demo 的目标是流程验证，不是实盘级收益承诺。**

## 五、常见隐患与陷阱

### 5.1 数据源层面

| 风险 | 说明 |
|---|---|
| 历史窗口短 | 免费接口通常只返回近期分钟数据 |
| 接口不稳定 | 某些 symbol 可能失败或格式不统一 |
| 非官方底库 | 更适合研究原型，不应直接视为生产级底库 |
| 夜盘差异 | 不同品种夜盘长度不同，需统一处理 |

### 5.2 期货特有风险

| 风险 | 说明 |
|---|---|
| 换月跳空 | 若直接拼裸价格，会严重污染趋势指标 |
| 涨跌停板 | 实盘中可能无法按模型价成交 |
| 保证金调整 | 杠杆与权益曲线会受交易所规则影响 |
| 合约退市 | 到期前后流动性迅速衰减 |
| 非连续交易时段 | 日夜盘切换会放大 gap 效应 |

### 5.3 回测口径风险

| 风险 | 说明 |
|---|---|
| 同 bar 成交 | 易产生未来函数偏乐观 |
| 主连黑箱 | 若直接用黑箱主连，可能看不到真实换月影响 |
| 成本过低 | 商品期货短周期策略对成本极敏感 |
| 只看单品种 | 容易过拟合某一品种特性 |

## 六、标准回测流程

### 6.1 推荐执行步骤

1. 选定品种，例如 `RB`；
2. 枚举交割月合约，例如 `RB2510/RB2601/RB2605/RB2610`；
3. 分别下载 5 分钟 K 线；
4. 做字段统一、去重、排序、空值检查；
5. 对每个交割月单独计算 `close.pct_change()`；
6. 使用收益率链式拼接成 `norm_close`；
7. 在 `norm_close` 上计算双均线；
8. 根据 `position.shift(1)` 计算策略收益；
9. 输出净值、回撤、交易表、指标 JSON；
10. 若要扩展为正式研究，再做参数扫描、样本外检验、多品种聚合。

### 6.2 适合继续扩展的方向

1. 从单品种扩展到 `RB / M / AU / CU / I / TA` 多品种；
2. 从自然月连接升级为 **流动性驱动换月**；
3. 加入波动率目标仓位；
4. 加入交易时段过滤；
5. 引入 ATR 止损 / 移动止损 / 趋势过滤；
6. 组合层聚合多品种权益曲线。

## 九、结论

在“**免费方案 + 5 分钟 K 线 + 分到期合约 + 连续化 + 时序 CTA**”这一约束下，本次推荐的落地路径是：
